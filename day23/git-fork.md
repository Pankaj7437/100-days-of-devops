# 🚀 Fork Git Repository (GUI)

## 📌 Overview

This task involves forking an existing Git repository from one user account to another using a graphical interface.

---

## 🧠 Problem Statement

- Source repository: `sarah/blog`
- Target user: `jon`
- Objective: Create a fork of the repository under `jon`'s account

---

## 🛠️ Steps to Perform

1. Login as user `jon`
2. Navigate to the repository:
```

sarah/blog

```
3. Click on the **Fork** button (top-right corner)
4. Select the destination account:
```

jon

```
5. Wait for the fork process to complete

---

## ✅ Verification

- New repository created:
```

jon/blog

```
- Fork relationship visible:
```

forked from sarah/blog

```

---

## 🎯 Key Learnings

- Difference between fork and clone
- Repository ownership transfer via forking
- GUI-based Git operations
- Working with multiple user accounts

---

## 🔥 DevOps Insight

> Forking allows independent development without affecting the original repository.
